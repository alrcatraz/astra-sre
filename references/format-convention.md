# astra-sre 格式约定

> 最后更新: 2026-06-07
> 原则：工具强制的→听工具的；自己选择的→随你喜欢
> 详见 Fact Store tag=format

## 一般工作 / 基础设施

| 用途 | 用 | 理由 |
|:----|:--|:-----|
| 配置文件 | **YAML** | 有注释，嵌套自然，Hermes/Docker/CI 生态原生 |
| 数据交换 / API | **JSON** | 标准、通用、所有语言原生解析 |
| Python 包元数据 | **TOML** | `pyproject.toml` PEP 621 标准，没得选 |
| Excel 导入 | **CSV** | Excel 原生打开 |

## C# / Godot 项目

| 用途 | 用 | 理由 |
|:----|:--|:-----|
| `.csproj` | **XML** | MSBuild 强制格式，没得选 |
| Unity UI Toolkit | **UXML**（XML） | Unity 原生格式 |
| 你自己的配置 | **你开心用啥用啥** | 你的项目你的选择 |

## 研究 / ML 实验

| 用途 | 用 | 理由 |
|:----|:--|:-----|
| 训练参数配置 | **YAML** | 和基础设施保持一致，有注释 |
| 实验日志 / 训练曲线 | **JSONL**（每行一个 JSON） | 可增量追加，每行独立解析 |
| 论文图表数据 | **CSV** | 导入 Origin / Excel / Matplotlib |

## 基本原则

- **人读的 → YAML**
- **机器读的 → JSON**
- **工具强制 → 听工具的**
- **自己选择 → 你说了算**

## 不推荐的格式

| 格式 | 原因 |
|:----|:-----|
| XML（自己可控的场景）| 除非你喜欢，否则比 YAML 多 40% token |
| INI | 表达能力有限，没有嵌套 |
